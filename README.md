# [Kiho]

**"비즈니스 가치를 이해하고, 탄탄한 아키텍처로 문제를 해결하는 개발자"**

현재 SI 환경에서 다양한 도메인의 문제를 해결하며 2년 차 개발자로 성장하고 있습니다. 
언어는 달라도 업무를 쉽고 명확하게 정의하기 위해서는 단순한 구현 뿐 아니라 **DDD(Domain-Driven Design), Hexagonal Architecture, FSD(Feature-Sliced Design)** 등 유지보수하기 좋은 구조가 필수라고 생각이 듭니다.
유지보수에 용이한 구조는 또한, AI에게 보다 효율적인 명령을 내릴 수 있게 해 주고 동일 토큰 대비 의도에 맞는 명확한 output을 끌어낼 수 있습니다. 
최근에는 아키텍쳐 구조에 중점을 두고 탄탄한 아키텍쳐 위에 AI와 LLM을 활용해 각 레이어를 채우는 빠르고 효율적인 프로젝트 설계에 깊은 관심을 두고 있습니다.
